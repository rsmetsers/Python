Voor Praktische Opdrachten (PO's) maakt deze cursus gebruik van PyCharm.

Programma draaien en foutmeldingen

Als je code hebt geschreven dat moet je deze runnen. Dat kan op verschillende manieren:

  1. Door op de groene driehoek (bovenin je scherm, links onderaan de scherm, of net links van je code) te klikken, of

  2. Door met je rechtermuisknop op task.py te klikken en "Run 'task'" te kiezen.

Als je correcte code hebt geschreven krijg je in de Python Console de melding 'Process finished with exit code 0'.

Als je een andere melding krijgt, dan heb je een fout in je programmacode gemaakt. Als je iets intikt dat geen juist Python code is zal de interpreter klagen. In de foutmelding staat bij in welke regel ('line ... ') de fout staat.

Je antwoord controleren

Het kan zijn dat je wel juist Python code hebt geschreven, maar dat jouw programma niet doet wat het hoort te doen. Controleer dus altijd jouw antwoord.

Als je wilt controleren of je de vraag goed beantwoord hebt, dan klik je op het blaadje met het vinkje, net onder 'Task Description'.

Controleer je uitwerking

Als je antwoord fout is, krijg je daarvan een melding. Herstel deze voordat je verder gaat.

Let op: Je moet dus jouw antwoord (laten) controleren voordat je verder gaat, anders telt die niet als gemaakt.

Tips

Kom je niet verder en wil je een aanwijzing? Klik dan op het lampje voor een tip. Als die er zijn, dan kan je met de pijltjes nog meer tips bekijken.

Opgave 2.1 Je eerste programma

Als je een nieuwe programmeer omgeving test begin je altijd eerst met een programma te schrijven dat "Hello, world!" afdrukt. Dit om te controleren dat alles juist ingesteld is en werkt. Dat programma schrijf je met de volgende code:

print( "Hello, world!" )

  1. Run het programma. Druk op het driehoekje.

  2. Controleer de uitvoer. Zie je dat de tekst “Hello, world!” getoond is in de Console onderaan je scherm?

  3. Controleer je antwoord. Druk op de knop met het vinkje onder 'Task Description'.

Zie je een groene ballon met 'Congratulations'? Dan is je opdracht goedgekeurd. Als je antwoord fout is, krijg je daarvan een melding. Herstel deze en controleer jouw antwoord nogmaals voordat je verder gaat.

Let op: Je moet dus jouw antwoord (laten) controleren voordat je verder gaat, anders telt die niet als gemaakt.
